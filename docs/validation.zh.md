# 驗證結果

這份文件彙整支撐 Absolute-Zero Heat Sink Bug 與補丁後熱模型行為的數值交叉驗證。

## 核心主張

`ThermalModel::startup()` 沒有初始化中間 RC 節點，讓它們停在 0 K。這會讓 Junction 在正功耗下反而降溫；將節點初始化到 ambient 後，行為恢復正常。

## 主要證據

- **五方比對驗證**：解析解（Analytical）、SPICE 模擬（ngspice）、Python Backward Euler 求解器、gem5 原始 bug 版本、gem5 修補版進行全方位物理與數學對齊。
- **誤差分析**：證明 bug 不是數值不穩定或步長誤差，而是物理層面的初始化錯誤（Bug 偏離幅度比數值誤差大 2 萬倍）。
- **收斂性分析**：timestep sweep 顯示 Backward Euler 收斂符合理論預期的 1 階精度。
- **長時間系統級運行**：55.116 秒的 patched Full-System 運行證明 Junction 溫度在各類複雜動態工作負載下均不再低於環境溫度。

## 結論

三條獨立證據鏈指向同一個根因：

1. **原始碼邏輯審計**：確認中間節點初始化遺漏。
2. **gem5 仿真實測**：觀測到異常降溫。
3. **獨立物理求解器重現**：Python BE 求解器重現 gem5 異常；SPICE 電路模擬驗證解析解。

這個 bug 與其 patch 因而都能被數學、軟體模擬與硬體等效電路三重驗證，達到了極高的學術與工業級可信度。
